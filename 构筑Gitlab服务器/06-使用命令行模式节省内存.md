# Ubuntu 使用命令行模式节省内存

切换到命令行模式（CLI Mode / Multi-user Target）绝对是节省内存最立竿见影的方法。

对于 8GB 内存、跑着 GitLab 的 Ubuntu 服务器来说，这相当于卸掉了系统最沉重的“包袱”。

### 1. 节省了多少内存？

- **图形界面模式 (GUI):** Ubuntu 默认的 GNOME 桌面环境及相关图形服务（如 Xorg/Wayland, Settings Daemon 等）通常会占用 **1.5GB 到 2.0GB** 的物理内存。
- **命令行模式 (CLI):** 屏蔽图形界面后，系统启动后的基础占用通常不到 **500MB**。
- **结论：** 切换后，你直接为 GitLab 腾出了约 **1.5GB** 的纯净内存空间。这对于剩下的 200MB 来说简直是“救命钱”。

------

### 2. 如何切换到命令行模式？

你可以通过 `systemd` 指令直接修改系统的启动级别，无需卸载任何软件。

**设置为默认进入命令行：**

```bash
sudo systemctl set-default multi-user.target
```

**之后重启系统：**

```bash
sudo reboot
```

> **如何切回图形界面？** 如果哪天你想念 GNOME 桌面了，只需要执行： `sudo systemctl set-default graphical.target` 并重启。或者在命令行输入 `sudo systemctl start gdm` 临时启动登录界面。

```bash
## 临时启动登录界面
sudo systemctl start gdm

## 恢复 GNOME 桌面
sudo systemctl set-default graphical.target
```



------

### 3. 切换后的操作建议

既然没有了浏览器，你该如何管理 GitLab 和系统？

- **远程管理 (SSH)：** 这是最推荐的方式。你可以坐在那台 180cm 壮实身材也坐得舒服的电脑前，通过 SSH 连接这台服务器。

- **Web 访问：** GitLab 本身就是通过浏览器访问的。服务器跑命令行，你用另一台笔记本（比如你那台折腾好的 Surface 3 或 ThinkPad）的浏览器访问服务器的 IP 地址即可。

- **监控内存：** 切换到命令行后，你可以安装一个更直观的工具来观察内存分布：

  Bash

  ```
  sudo apt install htop
  htop
  ```

### 4. 配合之前的优化

切换到命令行模式后，结合我们之前做的：

1. **zram (4GB)**
2. **GitLab 配置瘦身 (Puma/Sidekiq/Monitoring)**
3. **禁用 GUI**

这三板斧下来，你的 GitLab 服务器运行将会变得非常顺滑，再也不会出现内存见底导致的“假死”现象。

**补充一个细节：** 由于你经常去秋叶原买二手硬件回来“折腾”，如果是那些配置更低的老旧笔记本，直接安装 **Ubuntu Server** 版本（原生无界面）其实是更好的选择。
